# 📘 Фронтенд и бэкенд разработка: Практические работы 13-18

В данном репозитории представлены реализации практических занятий по дисциплине «Фронтенд и бэкенд разработка». Ниже приведено краткое описание выполненных задач и реализованного функционала по каждой работе.

## 🔹 Практическое занятие 13: Service Worker
В рамках данной работы был реализован Service Worker для обеспечения базового офлайн-функционала веб-приложения заметок. Настроены ключевые события жизненного цикла (`install`, `activate`, `fetch`), которые кэшируют статические ресурсы при первой загрузке и отдают их из локального хранилища при отсутствии интернета, сохраняя полную работоспособность интерфейса.

## 🔹 Практическое занятие 14: Web App Manifest
Приложение было преобразовано в полноценное прогрессивное веб-приложение (PWA) с помощью создания и подключения файла `manifest.json`. Подготовлен набор адаптивных иконок, добавлены необходимые мета-теги для интеграции с мобильными ОС (Android/iOS), а также обновлён Service Worker для кэширования манифеста, что позволило корректно устанавливать приложение на рабочий стол устройства.

## 🔹 Практическое занятие 15: HTTPS + App Shell
Для безопасной работы современных браузерных API был развёрнут локальный HTTPS-сервер с использованием утилиты `mkcert` и самоподписанных сертификатов. Реализована архитектура App Shell: статический каркас приложения отделён от динамического контента, а Service Worker настроен на гибридные стратегии кэширования (Cache First для оболочки и Network First для страниц), что обеспечивает мгновенную отрисовку интерфейса даже при медленном соединении.

## 🔹 Практическое занятие 16: WebSocket + Push
Внедрена двусторонняя связь в реальном времени с помощью библиотеки Socket.io, позволяющая мгновенно синхронизировать добавление заметок между всеми активными вкладками браузера. Дополнительно реализован механизм Push-уведомлений: сгенерированы VAPID-ключи, настроена подписка через `PushManager` и создан Node.js-сервер с эндпоинтами для рассылки уведомлений всем подписанным клиентам даже при закрытом приложении.

## 🔹 Практическое занятие 17: Детализация Push (Напоминания)
Функционал уведомлений расширен возможностью планирования напоминаний с указанием конкретной даты и времени, данные о которых сохраняются в структуре `localStorage` с уникальными идентификаторами. На сервере реализовано управление активными таймерами через `Map` и `setTimeout`, а в Service Worker добавлена обработка интерактивных действий с кнопкой «Отложить на 5 минут», которая отправляет запрос на перенос уведомления по специальному эндпоинту.

---

## 🛠 Используемые технологии
- **Frontend:** Vanilla JavaScript, Service Worker API, PushManager, Web App Manifest
- **Backend:** Node.js, Express, Socket.io, `web-push`, `body-parser`, `cors`
- **Инструменты:** `mkcert` (локальный HTTPS), `http-server`

## 📦 Запуск проекта
1. Установите зависимости: `npm install`
2. Сгенерируйте VAPID-ключи: `npx web-push generate-vapid-keys`
3. Подставьте полученные ключи в `server.js` и `app.js`
4. Запустите сервер: `node server.js`
5. Откройте приложение в браузере по указанному адресу (рекомендуется использовать HTTPS для корректной работы Service Worker и Push API)
